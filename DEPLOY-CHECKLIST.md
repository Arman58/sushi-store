# Деплой на прод — чеклист (релиз от 2026-07-07)

## Состав релиза
1. **Стоп-лист** — `Product.isAvailable`: бейдж «Нет в наличии», блокировка «+», кнопки на странице товара и в quick view; сервер отклоняет такие позиции в validate-cart и при оформлении. Тумблер «В наличии / Стоп-лист» в админ-таблице товаров.
2. **Недавно просмотренные** — секция на главной (localStorage, до 12 позиций).
3. **Отзывы: замыкание цикла** — блок «Как вам заказ?» на трекинге доставленного заказа со ссылками к отзывам блюд; раздел «Отзывы» в админке (фильтр по оценке, пагинация, удаление с пересчётом рейтинга).
4. **Корзина** — живой прогресс до минимальной суммы заказа (по зонам доставки), зелёное подтверждение при достижении.
5. **Mobile UX** — bottom-sheet товара (тап по карточке на ≤600px), stagger-анимация карточек меню, единые чипы категорий на главной и в /menu, один поиск (в шапке скрыт на /menu, `?search=` подхватывается), семантика (ul/li, button, aria-current/pressed, role=search).
6. **Полиш админки** — диалог подтверждения удаления товара вместо window.confirm.

## Порядок деплоя

### 1. Локально
```bash
git rm src/widgets/home/category-scroll.tsx   # мёртвый код (не используется)
npx prisma generate                            # обновит клиент (уберёт 8 ошибок tsc про isAvailable)
npx tsc --noEmit                               # 0 ошибок
npm run dev                                    # smoke-тест, п.3
```
Миграция релиза: `prisma/migrations/20260707100000_product_stop_list/` (одна аддитивная колонка) — применится на проде сама через `prisma migrate deploy` в build. Новых env нет.

### 2. Smoke-тест (mobile 320–430px в первую очередь)
- [ ] Меню: тап по блюду → bottom-sheet (фото, рейтинг, «Подробнее и отзывы», кнопка с ценой); блюдо с опциями → «Выбрать опции».
- [ ] Смена категории → карточки появляются каскадом; чипы категорий одинаковые на главной и /menu.
- [ ] На /menu нет поиска в шапке (ни mobile, ни desktop); поиск из шапки с главной ведёт на /menu и сразу фильтрует.
- [ ] Корзина: прогресс-бар до минималки растёт при добавлении, при достижении — зелёная галочка.
- [ ] Админка → Товары: тумблер «Стоп-лист» → на витрине бейдж «Нет в наличии», «+» неактивен, заказ с этой позицией отклоняется на чекауте.
- [ ] Заказ в статусе DONE → на трекинге блок «Как вам заказ?» → чип ведёт к секции отзывов.
- [ ] Админка → Отзывы: список, фильтр по звёздам, удаление.
- [ ] Десктоп: карточка по-прежнему открывает страницу товара (не шторку).

### 3. Откат
Код — revert/redeploy предыдущего билда. БД — миграция аддитивная, откат не нужен.
