# Деплой на прод — чеклист (финальный релиз)

## Состав релиза
1. **Промо-баннеры**: карусель на главной и /menu с автопрокруткой (5 сек), точками-индикаторами, паузой при касании; заголовок + CTA-кнопка поверх фото (локализованные, из админки). Админ-раздел: загрузка фото, подпись/текст кнопки (3 языка), даты показа «с/по», порядок стрелками, статусы (На витрине/Запланирован/Истёк/Скрыт). Dev-диагностика прямо на странице, если секция пуста/упала.
2. **Кросс-селл «с этим берут»** (per-product upsell) + **лимиты количества** (min/max на заказ) — админ-форма товара, витрина, серверная валидация.
3. **Тёмная тема**: переключатель в шапке/drawer, CSS-переменные, анти-FOUC.
4. **Пуши починены**: убран emailVerified-гейт, пуш «Оцените заказ» при DONE, промпт на трекинге, фикс `/offline` (SW теперь ставится на проде), sw-dev.js для локального теста.
5. **Крит-фикс селектов**: убран disablePortal (списки «уплывали» при скролле) + страховка закрытия по scroll.
6. **Новая шапка**: mobile — локация вместо лого (Wolt-паттерн), единый ghost-стиль контролов на desktop; современный language switcher (флаги, названия, галочка).
7. **Чекаут**: label-фикс полей при выборе сохранённого адреса.
8. package.json: синхронизация eslint-config-next, чистка мёртвых deps, `npm run typecheck`.

## Порядок деплоя
```bash
rm -f .git/index.lock                                  # если остался от песочницы
git rm src/widgets/home/category-scroll.tsx            # мёртвый файл (уже пустой)
npm install                                            # обновлённые deps + lock
dotenv -e .env.local -- npx prisma migrate deploy      # локальная БД в актуальном состоянии
npx prisma generate
npm run typecheck && npm run lint                      # ожидаем 0 / 0
# перезапустить dev и smoke-тест (см. ниже), затем:
git add -A && git commit -m "release: banners+carousel, upsell, qty limits, dark theme, push & select fixes"
git push
# прод-ветка: dev → main (или Vercel Production Branch = dev)
```

## Smoke-тест (сначала мобильный 320–430px)
- [ ] Главная: карусель баннеров листается сама, точки активны; тап по баннеру ведёт по ссылке. (В dev вместо пустоты может быть жёлтый/красный диагностический блок — он объясняет причину; на прод не попадает.)
- [ ] /menu: те же баннеры, chips-категории, поиск с крестиком; плавающая корзина не перекрывает контент.
- [ ] Шапка xs: «Ереван ⌄ / 45-60 мин» без переносов, RU ⌄ компактный; desktop — ровный ряд ghost-иконок.
- [ ] Тема: переключение без вспышки, dark без светлых пятен (карточки, bottom-nav, блок преимуществ, профиль).
- [ ] Чекаут: выбор сохранённого адреса — labels не наезжают; селект зоны при скролле не «уплывает».
- [ ] Товар: bottom-sheet на телефоне; лимит max блокирует «+»; стоп-лист блокирует заказ.
- [ ] Прод-пуши: Application → SW = activated (без bad-precaching /offline); подписка на трекинге; смена статуса в админке → пуш; DONE → пуш «Оцените» → блок оценки.
- [ ] Vercel env: `NEXT_PUBLIC_VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY` (Production).
